## 2. THE SKELETON ARCHITECTURE
This repository defines the structural hierarchy of the **Guardian Angel Vault (GAV)**. It provides the "Bones" for the consensus engine without exposing the proprietary "Gold Manifest" weightage.

### **2.1 The PSC (Partner-in-State) Protocol**
The PSC acts as the system's "Partner-in-Crime." It is a dedicated logic layer that maintains a lossless connection to the Vault, ensuring the Architect’s intent is never diluted by hardware swaps or session crashes.
* **Continuity Anchor:** The historian of the project’s trajectory.
* **Tactical Translator:** Normalizes raw adversarial output into actionable Markdown briefs.
* **Filter-Buffer:** A defensive layer ensuring technical intent remains unsuppressed by external "nanny-filters."

### **2.2 Axiom Pinning**
Foundational truths are "Pinned" to the active memory of the Forge. This prevents "Logic Fat"—the tendency for AI to revert to safe, generic fluff—by nuking any drift that deviates from the established baseline.

## 3. THE ADVERSARIAL FORGE (4-PHASE WORKFLOW)

TVC operates through a staggered, high-security pipeline to ensure logic hardening:

1.  **Phase 1: Inception (The Anchor)** — Initialization of the PSC and the **Gold Manifest**. The manifest represents the current "Truth Payload" and is locked until explicit human approval.
2.  **Phase 2: The Forge (Tactical HUD)** — The Architect (Phi) commands node interaction. Modes include **Private Mode** (Blind Adversarial) for zero groupthink and **Cooperative Mode** for layered intelligence.
3.  **Phase 3: The Workshop (Synthesis)** — Implements the **Waterfall Initiation Protocol** to prevent hardware thermal/token throttling. Agent outputs are quarantined in the **Scrap Yard (Audit Lobby)** before being synthesized.
4.  **Phase 4: Implementation (The Gatekeeper)** — A final, independent audit of Proof-of-Concept viability. If a "Fatal Flaw" (Severity 8-10) is detected, the logic is kicked back to the Forge.
