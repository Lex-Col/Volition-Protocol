## 1. Abstract
The Volition Protocol introduces a novel **Multi-Agent Context-Sharded Consensus (MACSC)** framework. Originally conceived during the development of GAP, it leverages an **Architect-Governed Recursive Forge** where a human user and a persistent **Partner-in-State** utilize parallel AI audits to hammer raw concepts into hardened logic through a discretionary, collaborative feedback loop.

## 2. The 4-Phase Workflow

### Phase 1: Inception (The Anchor - One-Time Setup)
The User establishes the baseline and selects a **Partner-in-State (PSC)**. High-end frontier models are recommended for security-critical logic.

### Phase 2: The Forge (Adversarial Roster Governance - RECURSIVE)
The draft is broadcast to a pool of **Adversarial Nodes (SANs)**. The User maintains absolute, at-will control over the Forge configuration at the start of **every** round:

- **Elastic Roster Scaling (At-Will Volume):** The User can expand or contract the number of nodes on the fly. There are no fixed limits or forced progressions. You can start with 5 nodes, drop to 1 for precision, and surge to 10+ for a final stress-test at any time.
- **Roster Management Modes:**
    - **The Static Squad:** Retain the same models to leverage **Individualized Recursive Memory (IRM)**—letting nodes track their own specific audit history.
    - **The Chaos Shuffle:** Randomize node selection each round to eliminate "Model Bias" and introduce fresh perspectives.
    - **The Specialist Swap:** Manually interchange models to bring in discipline-specific expertise (e.g., swapping a generalist for a kernel expert).
- **The Coverage Sync (At-Will):** Toggle between **Stealth** (Independent) and **Synced** (Collaborative - Current Round Only) modes to manage redundancy and find "Negative Space."

### Phase 3: The Workshop (Human-PSC Partnership - RECURSIVE)
The User and the **PSC (Partner-in-State)** sift through results in a 5-step cycle:
1. **The Broadcast:** Metadata-tagged critiques are gathered.
2. **The PSC Audit:** The PSC pre-sorts input into **Gold**, **Trash**, and **Glitch**.
3. **The Strategic Brief:** The PSC highlights conflicts and suggests hardenings.
4. **Authorization:** The User provides the "Write" command.
5. **The Re-Forge:** The PSC generates the **vNext** draft and updates the **Changelog of Intent**.

### Phase 4: Implementation (The Green Light)
The User exits the loop once **Stochastic Parity** is reached.

## 3. Architecture: The Anchor & The Forge

### 3.1 Primary State-Controller (PSC) - The Partner-in-State
The PSC is the **User's Right Hand**. It is a non-autonomous advisor that maintains the full **Institutional Memory** of the project and bridges technical knowledge gaps.

### 3.2 Individualized Recursive Memory (IRM) - The Forge
The Forge consists of rotating high-echelon LLMs. 
- **Temporal Isolation:** In "Synced Mode," nodes only see current-cycle data.
- **Thread Persistence:** If a node is retained, it maintains its thread (IRM). If swapped or added, it enters as a "Stateless" participant to ensure zero historical bias.
